# 관리자 화면 PRD (Product Requirements Document)

## 1. 화면 개요

### 1.1 화면명
관리자 화면 (Admin Screen)

### 1.2 화면 목적
관리자가 주문 현황을 모니터링하고, 재고를 관리하며, 주문을 처리할 수 있는 관리자 대시보드 화면

### 1.3 화면 구성
- 헤더 영역 (브랜드 로고, 네비게이션)
- 관리자 대시보드 영역 (주문 통계)
- 재고 현황 영역
- 주문 현황 영역

## 2. 화면 레이아웃 및 구성 요소

### 2.1 헤더 영역 (상단)
**위치**: 화면 최상단
**구성 요소**:
- **브랜드 로고**: 좌측 상단에 "COZY" 텍스트
- **네비게이션 버튼**: 우측 상단
  - "주문하기" 버튼 (비활성화 상태)
  - "관리자" 버튼 (현재 활성화된 화면 표시, 다크 테두리)

### 2.2 관리자 대시보드 영역 (상단 중간)
**위치**: 헤더 하단
**구성 요소**:

#### 2.2.1 섹션 제목
- "관리자 대시보드" 텍스트

#### 2.2.2 주문 통계 카드
다음 4개의 통계 항목을 가로로 나열:
1. **총 주문**: "총 주문 1"
2. **주문 접수**: "주문 접수 1"
3. **제조 중**: "제조 중 0"
4. **제조 완료**: "제조 완료 0"

### 2.3 재고 현황 영역 (중간)
**위치**: 관리자 대시보드 하단
**구성 요소**:

#### 2.3.1 섹션 제목
- "재고 현황" 텍스트

#### 2.3.2 재고 관리 카드들
3개의 상품별 재고 관리 카드가 가로로 배치:

**카드 1: 아메리카노 (ICE)**
- 상품명: "아메리카노 (ICE)"
- 현재 재고: "10개"
- 재고 조절 버튼: "+" 버튼과 "-" 버튼

**카드 2: 아메리카노 (HOT)**
- 상품명: "아메리카노 (HOT)"
- 현재 재고: "10개"
- 재고 조절 버튼: "+" 버튼과 "-" 버튼

**카드 3: 카페라떼**
- 상품명: "카페라떼"
- 현재 재고: "10개"
- 재고 조절 버튼: "+" 버튼과 "-" 버튼

### 2.4 주문 현황 영역 (하단)
**위치**: 화면 하단
**구성 요소**:

#### 2.4.1 섹션 제목
- "주문 현황" 텍스트

#### 2.4.2 주문 목록
현재 예시 주문:
- **주문 시간**: "7월 31일 13:00"
- **주문 상품**: "아메리카노(ICE) x 1"
- **주문 금액**: "4,000원"
- **액션 버튼**: "주문 접수" 버튼 (우측 정렬)

## 3. 기능 요구사항

### 3.1 주문 통계 모니터링 기능
- 실시간 주문 현황 통계 표시
- 주문 상태별 카운트 (총 주문, 접수, 제조 중, 완료)
- 통계 데이터의 실시간 업데이트

### 3.2 재고 관리 기능
- 각 상품별 현재 재고 수량 표시
- 재고 증가/감소 버튼을 통한 재고 조절
- 재고 변경 시 실시간 반영
- 재고 부족 시 알림 기능 (향후 구현)

### 3.3 주문 처리 기능
- 대기 중인 주문 목록 표시
- 주문 상세 정보 확인 (시간, 상품, 수량, 금액)
- 주문 접수 처리 기능
- 주문 상태 변경 기능

### 3.4 네비게이션 기능
- 주문하기 화면으로 이동
- 관리자 화면 내에서의 섹션 간 이동

## 4. 사용자 인터랙션

### 4.1 재고 관리 플로우
1. 관리자가 재고 현황 섹션에서 상품 확인
2. "+" 버튼 클릭으로 재고 증가
3. "-" 버튼 클릭으로 재고 감소
4. 변경된 재고 수량 즉시 반영

### 4.2 주문 처리 플로우
1. 관리자가 주문 현황 섹션에서 대기 주문 확인
2. 주문 상세 정보 검토
3. "주문 접수" 버튼 클릭
4. 주문 상태 변경 및 통계 업데이트

### 4.3 대시보드 모니터링 플로우
1. 관리자 대시보드에서 전체 주문 현황 파악
2. 각 상태별 주문 수량 확인
3. 필요시 해당 섹션으로 이동하여 상세 처리
