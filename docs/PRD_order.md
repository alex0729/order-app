# 주문하기 화면 PRD (Product Requirements Document)

## 1. 화면 개요

### 1.1 화면명
주문하기 화면 (Order Screen)

### 1.2 화면 목적
사용자가 커피 메뉴를 조회하고, 옵션을 선택하여 장바구니에 담고 최종 주문을 완료할 수 있는 메인 주문 화면

### 1.3 화면 구성
- 헤더 영역 (브랜드 로고, 네비게이션)
- 메뉴 상품 목록 영역
- 장바구니 영역

## 2. 화면 레이아웃 및 구성 요소

### 2.1 헤더 영역 (상단)
**위치**: 화면 최상단
**구성 요소**:
- **브랜드 로고**: 좌측 상단에 "COZY" 텍스트 (다크 그린 테두리)
- **네비게이션 버튼**: 우측 상단
  - "주문하기" 버튼 (현재 활성화된 화면 표시)
  - "관리자" 버튼 (비활성화 상태)

### 2.2 메뉴 상품 목록 영역 (중간)
**위치**: 헤더 하단, 장바구니 상단
**레이아웃**: 가로 3열 그리드 형태
**구성 요소**:

#### 2.2.1 상품 카드 구조
각 상품 카드는 다음 요소들을 포함:

1. **상품 이미지**
   - 위치: 카드 상단
   - 형태: 직사각형 이미지 영역 (현재는 플레이스홀더)
   - 크기: 카드 너비에 맞춤

2. **상품명**
   - 아메리카노(ICE): "아메리카노(ICE)"
   - 아메리카노(HOT): "아메리카노(HOT)"
   - 카페라떼: "카페라떼"

3. **기본 가격**
   - 아메리카노(ICE): "4,000원"
   - 아메리카노(HOT): "4,000원"
   - 카페라떼: "5,000원"

4. **상품 설명**
   - 텍스트: "간단한 설명..."
   - 목적: 상품에 대한 간단한 설명 제공

5. **추가 옵션**
   - "샷 추가 (+500원)" 체크박스
   - "시럽 추가 (+0원)" 체크박스
   - 각 옵션은 독립적으로 선택 가능

6. **담기 버튼**
   - 텍스트: "담기"
   - 스타일: 회색 버튼
   - 기능: 선택된 옵션과 함께 상품을 장바구니에 추가

### 2.3 장바구니 영역 (하단)
**위치**: 화면 하단
**구성 요소**:

#### 2.3.1 장바구니 헤더
- 제목: "장바구니"

#### 2.3.2 장바구니 아이템 목록
현재 예시 아이템:
1. **아메리카노(ICE) (샷 추가) X 1**
   - 가격: 4,500원 (기본 4,000원 + 샷 추가 500원)

2. **아메리카노(HOT) X 2**
   - 가격: 8,000원 (4,000원 × 2개)

#### 2.3.3 총 금액 및 주문 버튼
- **총 금액**: "총 금액 12,500원" (우측 정렬, 굵은 글씨)
- **주문하기 버튼**: "주문하기" (회색 버튼)

## 3. 기능 요구사항

### 3.1 메뉴 조회 기능
- 상품 목록을 카드 형태로 표시
- 각 상품의 기본 정보 (이름, 가격, 설명) 표시
- 상품 이미지 표시 (현재는 플레이스홀더)

### 3.2 옵션 선택 기능
- 각 상품별로 추가 옵션 제공
- 옵션별 추가 가격 표시
- 다중 옵션 선택 가능
- 옵션 선택 상태 시각적 표시

### 3.3 장바구니 관리 기능
- 상품을 장바구니에 추가
- 장바구니 아이템 목록 표시
- 각 아이템의 수량 및 총 가격 표시
- 장바구니 총 금액 계산 및 표시

### 3.4 주문 처리 기능
- 장바구니 내용을 바탕으로 주문 생성
- 주문 완료 후 적절한 피드백 제공

## 4. 사용자 인터랙션

### 4.1 상품 선택 플로우
1. 사용자가 원하는 상품 카드 확인
2. 추가 옵션 선택 (선택사항)
3. "담기" 버튼 클릭
4. 장바구니에 상품 추가 확인

### 4.2 장바구니 관리 플로우
1. 장바구니 영역에서 추가된 상품 확인
2. 총 금액 확인
3. "주문하기" 버튼 클릭으로 주문 완료

### 4.3 네비게이션 플로우
- "관리자" 버튼 클릭 시 관리자 화면으로 이동
